# Report Structure Rules

Use this reference when creating or checking a student report.

## Minimal Structure

For most practical and laboratory reports:

1. Protected title page.
2. `–°–ě–Ē–ē–†–Ė–ź–Ě–ė–ē` when required by the teacher or when the report is long enough to need it.
3. `–í–í–ē–Ē–ē–Ě–ė–ē` or a goal statement.
4. Main numbered sections based on the assignment.
5. `–ó–ź–ö–õ–ģ–ß–ē–Ě–ė–ē`.
6. `–°–ü–ė–°–ě–ö –ė–°–ü–ě–õ–¨–ó–ě–í–ź–Ě–Ě–ę–• –ė–°–Ę–ě–ß–Ě–ė–ö–ě–í` when sources are used or required.
7. `–ü–†–ė–õ–ě–Ė–ē–Ě–ė–Į` when screenshots, large tables, source code, or supplementary materials do not fit the main body.

## Practical Or Laboratory Report

Recommended sections:

- Purpose of work.
- Tasks.
- Initial data or variant.
- Practical part or execution steps.
- Results.
- Conclusion.
- Sources, if used.
- Appendices, if needed.

## Course Work

Recommended sections:

- Title page.
- Assignment details if required.
- Abstract or annotation if required.
- `–°–ě–Ē–ē–†–Ė–ź–Ě–ė–ē`.
- `–í–í–ē–Ē–ē–Ě–ė–ē`.
- Theoretical or analytical section when required.
- Project, practical, experimental, or calculation section.
- Results and evaluation.
- `–ó–ź–ö–õ–ģ–ß–ē–Ě–ė–ē`.
- Bibliography.
- Appendices.

## Section Logic

- The goal in the introduction must match the conclusion.
- Tasks should be concrete and verifiable.
- Do not end a section with a figure, table, or list.
- Insert explanation before figures and tables.
- Do not add theory unless the assignment or guide requires it.
- Use meaningful section headings, not generic `–•–ĺ–ī —Ä–į–Ī–ĺ—ā—č`.
